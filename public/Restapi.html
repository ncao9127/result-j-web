RestAPI for Results Jntuh

About

This project aims to provide students of JNTUH University with an efficient way of accessing their own semester results as well as those of their classmates. With just a single click, students will be able to view their academic performance throughout the semesters.

Authentication

The API currently does not require any form of authentication. Make a request with a hall ticket, it will provide the response.

API Endpoints

1. Get Student Results

Endpoint: /api/single

Method: GET

Description: Retrieves the semester results for a specific student.

Parameters:

  • student_roll_number (required): The unique identifier of the student.

Example Request:

                GET /api/single?htno=19361A0555

2. Get Notifications

Endpoint: /api/notifications

Method: GET

Example Request:

                GET /api/notifications

3. Get Student Semester Result

Endpoint: /api/single

Method: GET

Description: Retrieves the semester result for a specific student.

Parameters:

  • student_roll_number (required): The unique identifier of the student.

Example Request:

                GET /api/single?htno=19361A0555&code=4-1

3. Get Students Overall Results

Endpoint: /api/single

Method: GET

Description: Retrieves the Overall semester results for a specific students.

Parameters:

  • student_roll_number (required): The unique identifier of the student.

Example Request:

                GET /api/single?htno=19361A0555,20365A0504

Error Handling

The API follows standard HTTP error status codes. In case of an error, an appropriate error code will be returned along with a response containing an error message.

Example Error Response:

HTTP/1.1 500
Content-Type: application/json

{
  "error": "500 - Internal Server Error"
}
